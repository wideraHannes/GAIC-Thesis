Annotators label a sentence as argumentative if it expresses a claim or a premise that participates in a support or attack relation; purely descriptive, narrative, or organizational sentences are non-argumentative. Treat statements as argumentative when they make a debatable assertion or give a reason for/against such an assertion, even without explicit indicators; indicators like “therefore,” “because,” or “however” help but are not required. Questions, topic introductions, summaries of structure (e.g., “I will discuss…”), and recommendations for future action are non-argumentative. Exclude shell language (e.g., “I believe,” “for example”) unless it contains a meaningful negation; only the content-bearing statement is annotated. Split sentences into multiple components only if one statement is a reason for another; coordinated statements listing multiple reasons without inference remain a single component. Treat intermediate results in reasoning chains as premises, not claims, even if claim indicators appear. Boundary examples: (1) “What would happen if cloning were allowed?” → Non-argumentative (question, no asserted position). (2) “Therefore, human cloning should be prohibited.” → Argumentative (claim; conclusion signaled by indicator). (3) “I strongly disagree with this view.” → Non-argumentative unless the disagreed-with proposition is explicitly stated; otherwise no content to annotate.
